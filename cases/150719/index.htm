SEO-кейс: как мы увеличили региональный трафик на 1734%

Как благодаря простой настройке региональных поддоменов увеличить целевой трафик на сайт. Рассказываем на примере магазина верхней одежды.

Дмитрий Андреев, webit

Поделиться

Анализируем рынок

Одно из приоритетных направлений «Московской Меховой Компании» — увеличение продаж в регионах. Сайт нашего клиента не имел региональных представительств, но поисковой трафик из регионов составлял 40%.

Мы решили, что создание региональных поддоменов станет хорошим драйвером роста посещаемости. Таким образом мы планировали улучшить видимость в поиске в этих регионах.

На первом этапе следовало определиться со списком приоритетных по трафику регионов. Для этого использовали статистику спроса по основным товарным направлениям сайта. В работе были данные из 95 регионов РФ, полученные из Wordstat.

В результате выбрали 24 города с наибольшими показателями WS.

Создаём региональные поддомены

Теория

«Яндекс» даёт следующие рекомендации:

  • Чтобы пользователю было проще идентифицировать региональность поддомена сайта, добавьте в адрес название соответствующего региона (например, spb.example.com).
  • Для учёта региональности поддомена укажите на сайте полную контактную информацию организации в этом регионе и добавьте региональное представительство в «Яндекс.Справочник».
  • Региональный поддомен должен содержать информацию, полезную для пользователей именно из того региона, для которого он создан.
  • Региональный поддомен должен быть включен в навигацию по основному сайту, чтобы посетитель перешёл с главной страницы на страницу, которая относится к его региону.

Практика

На деле наш алгоритм был такой:

1. Создаём на сервере поддомены вида: kazan.mosmexa.ru и perm.mosmexa.ru.

2. Для каждого поддомена создаём отдельный файл robots.txt., в качестве host указываем поддомен:

3. Для каждого поддомена добавляем уникальные региональные контактные данные:

  • страницу с подробными контактами в конкретном регионе;

  • местные контакты в футере и хедере на всех страницах;

  • упоминание топонима и телефона в текстах.

4. Прописываем индивидуальные метатеги, заголовки, тексты — или как минимум создаём уникальные для каждого поддомена шаблоны с использованием следующих переменных:

  • {город в им.п.}
  • {город в пр.п.}
  • {региональный телефон}

5. Для каждого поддомена создаём отдельную организацию в «Яндекс.Справочнике». Наконец, проверяем, корректно ли присвоен регион в «Яндекс.Вебмастере».

Результат

Для анализа мы взяли данные по визитам из поиска (с вычетом бренда) за ноябрь-декабрь 2015-го (поддоменов не было) и ноябрь-декабрь 2016 года (уже с поддоменами).

Данные из «Яндекс.Метрики» — ссылка на полную таблицу

На всём сайте трафик вырос на 694,09%, в регионах с поддоменами — на 1734,28%, а без поддоменов — всего на 259,57%.

Чтобы исключить ошибку, мы также учитывали рост спроса в данной тематике. Для этого анализировали историю изменения WS по всем разделам, представленным на сайте. Например:

Так легко вычислить, на сколько процентов изменилась сумма всех WS. В нашем случае спрос вырос примерно на 28%. Учитывая новые данные, мы заново оценили трафик из регионов с поддоменами. Получилось, что он вырос на 1335,16%.

Скриншот из «Яндекс.Метрики» наглядно демонстрирует рост числа визитов из регионов с поддоменами.

Вывод

Для компаний, работающих с доставкой по всей России, важно отслеживать географию своих клиентов и заботиться, чтобы сайт был хорошо представлен в поиске в разных регионах.

Мы рекомендуем такой алгоритм действий по созданию региональных поддоменов:

  1. Оцените спрос в разных регионах и определитесь со списком поддоменов.
  2. Подготовьте местные контактные данные.
  3. Добавьте на каждый поддомен контакты, упоминание топонимов, собственный robots.txt.
  4. Присвойте каждому поддомену регион в «Яндекс.Вебмастере» и организацию в «Яндекс.Справочнике».
